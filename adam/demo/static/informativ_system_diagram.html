5-Level Cascade
Bilateral evidence fusion
›
L1
Archetype Prior
Cold-start priors from 8 archetypes. Each archetype carries differentiated Beta(α,β) priors across 10 persuasion mechanisms — not flat priors.
Prior ~ Beta(α_archetype, β_archetype)
Archetypes: Analytical Skeptic, Impulse Seeker, Brand Loyalist, Value Hunter, Social Validator, Quality Purist, Trend Follower, Need-Driven.
L2
Category Posterior
Category-specific evidence updates the archetype prior. Cross-category transfer propagates learnings from related categories with decay.
Posterior = Beta(α + Σsuccess_cat, β + Σfail_cat)
L3
Bilateral Edges (Core)
The prediction power. 7-dimension alignment edge between buyer psychology and seller messaging:
Need–Detect
Emo–Align
Cogn–Match
Trust–Transfer
Social–Proof
Urgency
Value–Frame
∂P(conversion) / ∂dimension
IV = P(signal) × Δ(accuracy) × PV(future)
L3 fully overrides L1/L2 when edge evidence exists. The edge is the signal.
L4
Inferential Transfer
Zero-shot transfer from ad creative profile. Uses construct-level causal graph to infer likely effectiveness on unseen buyer segments.
P(conv | no_edge) ≈ f(ad_profile, construct_graph)
L5
Future Extensions
Temporal dynamics, real-time attention signals, cross-device identity resolution. Planned.

Learning Loop — Conversion Event → 12 Update Systems
1
Conversion Event
Webhook from StackAdapt
›
StackAdapt fires conversion webhook with:
event_id, segment_id, conversion_type, revenue, timestamp, decision_id
2
HMAC Verify
SHA-256 signature
›
HMAC-SHA256(secret, body)
Validates webhook authenticity. Rejects if signature mismatch.
3
Dedup
event_id check
›
Redis set-based deduplication. If event_id seen within 48h window, skip processing. Prevents double-counting.
4
Map
decision_id → context
›
Maps conversion event back to original decision_id. Retrieves the full decision context from Redis (30 cached fields).
5
Retrieve Context
Full decision snapshot
›
Restores: activated mechanisms, weights, buyer profile, cascade state, page context, counterfactuals, gradient field. Everything needed for credit attribution.
6
12 Learning Systems
Parallel updates ↓
›
Click cards below for details on each system.
#1
Thompson Sampling
Updates Beta posteriors for each mechanism.
Beta(α+1, β) on success
Beta(α, β+1) on failure
Buyer-aware exploration: cold buyers get 3× bonus.
#2
Meta-Orchestrator
Updates strategy-level weights. Which combination of mechanisms works best for this (archetype, category) cell. EWA with decay.
#3
Neo4j Attribution
Writes OUTCOME edges in the graph connecting Decision → Mechanism → Result. Enables Shapley-value credit attribution across mechanism interactions.
#4
Graph Rewriter
Adjusts rule effectiveness scores based on outcomes. Rules that consistently underperform get deprioritized. Updates edge weights in the construct graph.
#5
Learning Hub
Distributes outcome signal to 30 Atom-of-Thought modules. Each atom updates its internal parameters based on whether its contribution was credit-positive.
#6
ML Ensemble
Online model weight adjustment. Gradient update for neural components; EWA for ensemble weights. Retrained on rolling 7-day window.
#7
Theory Learner
Construct-level causal learning. Updates theory graph edges: which psychological constructs causally predict conversion in this context.
#8
DSP Learning
Signal reliability scoring. Tracks which StackAdapt signals (segment, geo, device) are predictive vs noisy. Adjusts signal weights.
#9
Cognitive Learning
Updates alignment matrices between cognitive styles and mechanism effectiveness. Learns which processing modes favor which persuasion approaches.
#10
Page Context Learning
Learns mechanism × context interactions. "Scarcity works 2.1× better on high-cognitive-load pages." Updates context modulation weights.
#11
Mechanism Interactions
Portfolio covariance matrix updates. Learns which mechanism pairs are synergistic vs antagonistic. Updates MPT optimization constraints.
#12
Buyer Profile
Reduces buyer uncertainty across 20 dimensions. Well-characterized buyers (20+ interactions) get exploration bonus reduced to 0.5×. Redis 90-day TTL.
