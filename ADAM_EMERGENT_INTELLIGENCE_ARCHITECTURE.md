# Philosophical Foundation: What ADAM Actually Is

## The Category Error Everyone Makes

Most AI systems fall into one of two categories:

**LLM-Centric Systems**: Everything flows through the language model. The LLM is the "brain," other components are appendages. Intelligence resides in one place.

**ML-Pipeline Systems**: Features go in, predictions come out. Statistical patterns are learned from data. Intelligence is distributed but siloed.

ADAM is neither. **ADAM is a cognitive ecosystem** where:

1. **Claude provides theoretical intelligence** — psychological theory, mechanism understanding, causal reasoning about why humans do what they do
2. **Empirical pattern discovery provides statistical intelligence** — behavioral signatures, correlation detection, predictive patterns that emerge from data
3. **Nonconscious behavioral analytics provide implicit intelligence** — supraliminal signals (keystroke dynamics, mouse movements, scroll patterns) that reveal psychological states the user cannot articulate
4. **The graph database provides emergent memory** — relationships and patterns that none of the individual systems could produce alone

The critical insight: **Intelligence doesn't reside in any component. It emerges from their interplay.**

## The Three Principles That Make This Work

### Principle 1: Design for Discovery, Not Just Retrieval

Traditional systems optimize for efficient retrieval of known information. ADAM optimizes for **discovery of unknown relationships**.

This means:
- Graph schemas that DON'T over-constrain what relationships can exist
- Traversal patterns that find connections the system wasn't explicitly looking for
- Indexes that enable serendipitous pattern recognition
- Logging rich enough to enable post-hoc discovery of patterns we didn't anticipate

### Principle 2: Automated Learning Without Human Bottlenecks

Every piece of the system has access to every discovery. Learning happens in a fluid, automated way unrestricted by human oversight. The system is **aggressively trying to learn and improve**.

This means:
- Cross-component learning signals propagate in real-time
- Anomaly detection triggers automatic hypothesis generation
- Confidence-based routing determines what needs human review vs. automated action
- No "learning silos" where insights don't propagate

### Principle 3: Structural Moats Through Architecture

Competitive advantage emerges from **structural flywheels** that compound over time, not just better algorithms that can be copied.

This means:
- Every interaction makes the graph richer, which makes predictions better, which generates more interactions
- Psychological construct relationships that competitors cannot replicate by hiring more engineers
- Data assets that become more valuable as they grow, not just larger
- Network effects within the platform itself

# Part 1: Graph as Cognitive Medium

## 1.1 The Paradigm Shift: From Storage to Discovery

Traditional graph database usage:
```
Define schema → Store entities → Query known relationships → Return results
```

ADAM's cognitive medium approach:
```
Define flexible schema → Store entities with rich metadata → 
Enable emergent relationship discovery → Surface patterns no one looked for →
Feed discoveries back into schema evolution
```

### What This Means Architecturally

The Neo4j schema must be designed with **discovery affordances**—structural features that make unexpected findings possible:

1. **Rich Metadata on Everything**: Every node and relationship carries metadata that enables pattern mining even if we don't know what patterns we're looking for

2. **Permissive Relationship Types**: The schema allows relationships that weren't explicitly planned, enabling emergence

3. **Temporal Versioning**: All changes are tracked, enabling discovery of patterns in how relationships evolve

4. **Confidence and Evidence Tracking**: Every assertion includes its evidential basis, enabling meta-learning about what kinds of evidence predict accuracy

## 1.3 Discovery Queries: Finding What We Weren't Looking For

The power of the cognitive medium emerges from queries designed to surface unexpected patterns:

```cypher
// =============================================================================
// DISCOVERY QUERIES
// Finding patterns the system wasn't explicitly looking for
// =============================================================================

// ----- QUERY 1: Mechanism Interactions We Didn't Model -----
// Find mechanism pairs that co-activate more than expected
MATCH (u:User)-[r1:RESPONDS_TO]->(m1:CognitiveMechanism)
MATCH (u)-[r2:RESPONDS_TO]->(m2:CognitiveMechanism)
WHERE m1.mechanism_id < m2.mechanism_id  // Avoid duplicates
AND r1.success_rate > 0.7 AND r2.success_rate > 0.7
WITH m1, m2, count(u) as co_high_responders,
     avg(r1.success_rate + r2.success_rate) as combined_effectiveness
// Compare to expected based on individual rates
MATCH (m1)<-[r1all:RESPONDS_TO]-(:User)
MATCH (m2)<-[r2all:RESPONDS_TO]-(:User)
WITH m1, m2, co_high_responders, combined_effectiveness,
     avg(r1all.success_rate) as m1_base,
     avg(r2all.success_rate) as m2_base
WHERE co_high_responders > 100  // Statistical significance
AND combined_effectiveness > (m1_base + m2_base) * 1.1  // 10% synergy threshold
AND NOT EXISTS((m1)-[:INTERACTS_WITH]-(m2))  // Not already known
RETURN m1.name, m2.name, co_high_responders, 
       combined_effectiveness, m1_base + m2_base as expected,
       combined_effectiveness / (m1_base + m2_base) as synergy_ratio
ORDER BY synergy_ratio DESC
LIMIT 10

// ----- QUERY 2: Behavioral Signatures Without Construct Mapping -----
// Find behavioral patterns that predict outcomes but aren't explained by theory
MATCH (bs:BehavioralSignature)
WHERE bs.outcome_correlation > 0.25
AND bs.population_prevalence > 0.05
AND bs.is_mapped_to_construct = false
AND bs.nearest_constructs[0].distance > 0.4  // Not close to any construct
RETURN bs.signature_id, bs.pattern_definition, 
       bs.outcome_correlation, bs.population_prevalence,
       bs.nearest_constructs
ORDER BY bs.outcome_correlation DESC

// ----- QUERY 3: Construct-Mechanism Relationships That Theory Didn't Predict -----
// Find which constructs predict mechanism effectiveness but weren't theorized
MATCH (c:PsychologicalConstruct)-[p:PREDICTS_RESPONSE]->(m:CognitiveMechanism)
WHERE p.is_theoretically_predicted = false
AND p.correlation > 0.3
AND p.sample_size > 1000
RETURN c.name as construct, m.name as mechanism,
       p.correlation, p.effect_direction,
       p.theoretical_explanation
ORDER BY p.correlation DESC

// ----- QUERY 4: User Segments With Anomalous Patterns -----
// Find user clusters where mechanism effectiveness differs significantly from population
MATCH (u:User)-[r:RESPONDS_TO]->(m:CognitiveMechanism)
WITH u.psychological_cluster_id as cluster, m.mechanism_id as mechanism,
     avg(r.success_rate) as cluster_avg, count(u) as cluster_size
MATCH (:User)-[r_all:RESPONDS_TO]->(m_all:CognitiveMechanism {mechanism_id: mechanism})
WITH cluster, mechanism, cluster_avg, cluster_size,
     avg(r_all.success_rate) as population_avg,
     stdev(r_all.success_rate) as population_stdev
WHERE cluster_size > 100
AND abs(cluster_avg - population_avg) > 2 * population_stdev  // 2+ std devs
RETURN cluster, mechanism, cluster_avg, population_avg,
       (cluster_avg - population_avg) / population_stdev as z_score,
       cluster_size
ORDER BY abs(z_score) DESC

// ----- QUERY 5: Temporal Pattern Discovery -----
// Find relationships whose effectiveness has changed significantly
MATCH (rh:RelationshipHistory)
WHERE size(rh.significant_changes) > 0
AND rh.significant_changes[-1].date > datetime() - duration('P30D')
MATCH (u:User {user_id: rh.subject_id})-[r:RESPONDS_TO]->(m:CognitiveMechanism {mechanism_id: rh.object_id})
RETURN u.user_id, m.name, 
       rh.trend_direction, rh.trend_confidence,
       rh.significant_changes[-1] as recent_change
ORDER BY recent_change.magnitude DESC

// ----- QUERY 6: Emergent Construct Validation -----
// Check if emergent constructs are behaving like real psychological constructs
MATCH (ec:EmergentConstruct)
WHERE ec.requires_theoretical_review = true
// Check for stability (real constructs should be stable)
MATCH (u:User)-[r:HAS_SIGNATURE]->(ec)
WITH ec, avg(r.stability_score) as avg_stability, count(u) as user_count
// Check for predictive validity
MATCH (u:User)-[r:HAS_SIGNATURE]->(ec)
MATCH (u)-[o:HAD_OUTCOME]->(:ConversionEvent)
WITH ec, avg_stability, user_count,
     avg(o.success) as outcome_rate
// Check for discriminant validity (different from existing constructs)
MATCH (ec)-[d:DISTANCE_TO]->(pc:PsychologicalConstruct)
WITH ec, avg_stability, user_count, outcome_rate,
     min(d.distance) as min_construct_distance
WHERE avg_stability > 0.7  // Stable
AND outcome_rate > 0.5     // Predictive
AND min_construct_distance > 0.3  // Distinct
RETURN ec.construct_id, ec.defining_signals,
       avg_stability, user_count, outcome_rate, min_construct_distance,
       ec.auto_generated_hypotheses
ORDER BY outcome_rate DESC
```

```python
    async def _discover_mechanism_interactions(self) -> List[Discovery]:
        """Find mechanism pairs that interact unexpectedly."""
        
        query = """
        // Find mechanism pairs with synergistic effects
        MATCH (u:User)-[r1:RESPONDS_TO]->(m1:CognitiveMechanism)
        MATCH (u)-[r2:RESPONDS_TO]->(m2:CognitiveMechanism)
        WHERE m1.mechanism_id < m2.mechanism_id
        AND r1.success_rate > 0.7 AND r2.success_rate > 0.7
        
        WITH m1, m2, 
             collect(DISTINCT u.user_id) as shared_users,
             avg(r1.success_rate) as m1_when_both,
             avg(r2.success_rate) as m2_when_both
             
        WHERE size(shared_users) > $min_sample
        
        // Get baseline rates
        MATCH (m1)<-[r1base:RESPONDS_TO]-(:User)
        MATCH (m2)<-[r2base:RESPONDS_TO]-(:User)
        
        WITH m1, m2, shared_users,
             m1_when_both, m2_when_both,
             avg(r1base.success_rate) as m1_baseline,
             avg(r2base.success_rate) as m2_baseline
             
        // Check if not already known
        WHERE NOT EXISTS((m1)-[:INTERACTS_WITH]-(m2))
        
        // Calculate interaction effect
        WITH m1, m2, shared_users,
             (m1_when_both + m2_when_both) / 2 as combined_when_both,
             (m1_baseline + m2_baseline) / 2 as combined_baseline,
             size(shared_users) as sample_size
             
        WHERE combined_when_both > combined_baseline * 1.1  // 10%+ synergy
        
        RETURN m1.mechanism_id as mechanism_1,
               m2.mechanism_id as mechanism_2,
               m1.name as name_1,
               m2.name as name_2,
               combined_when_both,
               combined_baseline,
               combined_when_both / combined_baseline as synergy_ratio,
               sample_size
        ORDER BY synergy_ratio DESC
        LIMIT 10
        """
        
        async with self.driver.session() as session:
            result = await session.run(
                query, 
                min_sample=self.config.min_sample_size
            )
            records = await result.data()
        
        discoveries = []
        for record in records:
            # Statistical validation
            effect_size = record['synergy_ratio'] - 1.0
            
            if effect_size >= self.config.effect_size_threshold:
                # Generate hypotheses for this interaction
                hypotheses = await self._generate_hypotheses_for_interaction(
                    record['mechanism_1'],
                    record['mechanism_2'],
                    record['synergy_ratio'],
                    record['sample_size']
                )
                
                discovery = Discovery(
                    discovery_id=f"int_{record['mechanism_1']}_{record['mechanism_2']}",
                    discovery_type=DiscoveryType.MECHANISM_INTERACTION,
                    description=f"Mechanisms {record['name_1']} and {record['name_2']} show {effect_size:.1%} synergistic interaction",
                    evidence={
                        'mechanism_1': record['mechanism_1'],
                        'mechanism_2': record['mechanism_2'],
                        'synergy_ratio': record['synergy_ratio'],
                        'combined_effectiveness': record['combined_when_both'],
                        'baseline_effectiveness': record['combined_baseline']
                    },
                    sample_size=record['sample_size'],
                    statistical_significance=self._calculate_significance(
                        record['combined_when_both'],
                        record['combined_baseline'],
                        record['sample_size']
                    ),
                    effect_size=effect_size,
                    confidence=self._calculate_discovery_confidence(
                        effect_size, record['sample_size']
                    ),
                    requires_human_review=False,  # Calculated below
                    auto_action_approved=False,   # Calculated below
                    hypotheses=hypotheses,
                    discovered_at=datetime.utcnow()
                )
                
                # Route based on confidence
                discovery = self._route_discovery(discovery)
                discoveries.append(discovery)

    # -------------------------------------------------------------------------
    # Discovery Routing and Processing
    # -------------------------------------------------------------------------
    
    def _route_discovery(self, discovery: Discovery) -> Discovery:
        """
        Route discovery based on confidence:
        - High confidence: Auto-action approved
        - Medium confidence: System acts but logs for review
        - Low confidence: Requires human review
        """
        
        if discovery.confidence >= self.config.auto_action_confidence:
            discovery.auto_action_approved = True
            discovery.requires_human_review = False
        elif discovery.confidence >= self.config.human_review_threshold:
            discovery.auto_action_approved = True  # Act but track
            discovery.requires_human_review = True  # Flag for review
        else:
            discovery.auto_action_approved = False
            discovery.requires_human_review = True
        
        return discovery
    
    async def _process_discovery(self, discovery: Discovery):
        """Process a discovery - either auto-act or queue for review."""
        
        # Always persist the discovery
        await self._persist_discovery(discovery)
        
        if discovery.auto_action_approved:
            # Act on discovery automatically
            await self._act_on_discovery(discovery)
            
            # Propagate learning to all components
            await self._propagate_learning(discovery)
        
        if discovery.requires_human_review:
            # Queue for human review (but may have already acted)
            await self._queue_for_review(discovery)
    
    async def _act_on_discovery(self, discovery: Discovery):
        """Take automatic action based on discovery type."""
        
        if discovery.discovery_type == DiscoveryType.MECHANISM_INTERACTION:
            # Create the interaction relationship in the graph
            await self._create_mechanism_interaction(discovery)
            
        elif discovery.discovery_type == DiscoveryType.BEHAVIORAL_SIGNATURE:
            # Flag signature for inclusion in predictions
            await self._activate_signature_for_predictions(discovery)
            
        elif discovery.discovery_type == DiscoveryType.EMERGENT_CONSTRUCT:
            # Create emergent construct node and relationships
            await self._create_emergent_construct(discovery)
        
        # Start testing highest-confidence hypothesis
        best_hypothesis = max(
            discovery.hypotheses, 
            key=lambda h: h.confidence
        )
        await self._start_hypothesis_test(best_hypothesis)
        
        discovery.acted_upon_at = datetime.utcnow()
    
    async def _propagate_learning(self, discovery: Discovery):
        """
        Propagate discovery to ALL components that could use it.
        
        This is critical: learning must be fluid and unrestricted.
        Every component should benefit from every discovery.
        """
        
        # Emit event to Kafka for all subscribers
        event = {
            'event_type': 'discovery_made',
            'discovery_id': discovery.discovery_id,
            'discovery_type': discovery.discovery_type.value,
            'evidence': discovery.evidence,
            'confidence': discovery.confidence,
            'timestamp': datetime.utcnow().isoformat()
        }
        
        await self._emit_kafka_event('adam.discoveries', event)
        
        # Update hot caches immediately
        await self._invalidate_affected_caches(discovery)
        
        # Update relevant model priors
        await self._update_model_priors(discovery)
        
        # Trigger re-evaluation of affected users
        await self._queue_affected_user_reevaluation(discovery)
    
    # -------------------------------------------------------------------------
    # Statistical Helpers
    # -------------------------------------------------------------------------
    
    def _calculate_discovery_confidence(
        self,
        effect_size: float,
        sample_size: int
    ) -> float:
        """
        Calculate confidence in a discovery based on effect size
        and sample size.
        """
        # Bayesian-inspired confidence calculation
        # More data + larger effect = higher confidence
        
        effect_component = min(effect_size / 0.5, 1.0)  # Saturates at d=0.5
        sample_component = 1 - np.exp(-sample_size / 500)  # Saturates ~2000
        
        return effect_component * sample_component
    
    def _calculate_significance(
        self,
        observed: float,
        expected: float,
        sample_size: int
    ) -> float:
        """Calculate p-value for observed vs expected."""
        # Simplified - in production use proper statistical tests
        se = np.sqrt(expected * (1 - expected) / sample_size)
        z = (observed - expected) / se
        return 2 * (1 - stats.norm.cdf(abs(z)))
    
    def _correlation_significance(
        self,
        r: float,
        n: int
    ) -> float:
        """Calculate p-value for correlation coefficient."""
        t = r * np.sqrt((n - 2) / (1 - r**2))
        return 2 * (1 - stats.t.cdf(abs(t), n - 2))
    
    def _cluster_related_signatures(
        self,
        records: List[Dict]
    ) -> List[Dict]:
        """Cluster related signatures into potential constructs."""
        # Build graph of signature relationships
        from collections import defaultdict
        
        adjacency = defaultdict(set)
        signature_data = {}
        
        for record in records:
            sig1, sig2 = record['sig1'], record['sig2']
            adjacency[sig1].add(sig2)
            adjacency[sig2].add(sig1)
            signature_data[sig1] = record['pattern1']
            signature_data[sig2] = record['pattern2']
        
        # Find connected components (clusters)
        visited = set()
        clusters = []
        
        def dfs(node, cluster):
            visited.add(node)
            cluster.add(node)
            for neighbor in adjacency[node]:
                if neighbor not in visited:
                    dfs(neighbor, cluster)
        
        for sig in adjacency:
            if sig not in visited:
                cluster = set()
                dfs(sig, cluster)
                if len(cluster) >= 2:  # At least 2 signatures
                    clusters.append(cluster)
        
        # Convert to structured output
        result = []
        for cluster in clusters:
            cluster_records = [
                r for r in records 
                if r['sig1'] in cluster and r['sig2'] in cluster
            ]
            
            if cluster_records:
                result.append({
                    'signatures': list(cluster),
                    'patterns': {s: signature_data.get(s) for s in cluster},
                    'internal_correlation': np.mean([
                        r['signature_correlation'] for r in cluster_records
                    ]),
                    'outcome_prediction': np.mean([
                        r['combined_outcome_rate'] for r in cluster_records
                    ]),
                    'sample_size': min(r['shared_users'] for r in cluster_records),
                    'significance': 0.001  # Placeholder
                })
        
        return result
```

```python
"""
Learning Signal Propagation System

Every outcome generates learning signals that propagate to EVERY
component that could use them. No silos. No bottlenecks.
"""

from dataclasses import dataclass
from datetime import datetime
from typing import Dict, Any, List, Optional
from enum import Enum
import asyncio
from pydantic import BaseModel, Field
import aiokafka
from redis import asyncio as aioredis
from neo4j import AsyncGraphDatabase


class LearningSignalType(str, Enum):
    """Types of learning signals the system generates."""
    OUTCOME_OBSERVED = "outcome_observed"
    MECHANISM_ACTIVATED = "mechanism_activated"
    TRAIT_INFERRED = "trait_inferred"
    STATE_TRANSITIONED = "state_transitioned"
    DISCOVERY_MADE = "discovery_made"
    HYPOTHESIS_VALIDATED = "hypothesis_validated"
    CALIBRATION_UPDATED = "calibration_updated"
    CLUSTER_SHIFTED = "cluster_shifted"


@dataclass
class LearningSignal:
    """A learning signal that propagates through the system."""
    signal_id: str
    signal_type: LearningSignalType
    source_component: str
    timestamp: datetime
    
    # The learning payload
    payload: Dict[str, Any]
    
    # Routing information
    target_components: List[str]  # Empty = broadcast to all
    priority: str  # immediate, fast, async, batch
    
    # Tracking
    propagation_status: Dict[str, str] = None  # component → status


class LearningPropagator:
    """
    Propagates learning signals to all components.
    
    This is the nervous system of ADAM - ensuring that every
    insight reaches every component that could use it.
    """
    
    def __init__(
        self,
        kafka_bootstrap: str,
        redis_url: str,
        neo4j_uri: str,
        neo4j_auth: tuple
    ):
        self.kafka_producer = None
        self.redis = None
        self.neo4j = None
        
        self.kafka_bootstrap = kafka_bootstrap
        self.redis_url = redis_url
        self.neo4j_uri = neo4j_uri
        self.neo4j_auth = neo4j_auth
        
        # Component registry - all components that receive signals
        self.component_subscriptions: Dict[str, List[LearningSignalType]] = {
            'copy_generation': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.MECHANISM_ACTIVATED,
                LearningSignalType.DISCOVERY_MADE
            ],
            'cold_start': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.CLUSTER_SHIFTED,
                LearningSignalType.DISCOVERY_MADE
            ],
            'inference_engine': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.MECHANISM_ACTIVATED,
                LearningSignalType.CALIBRATION_UPDATED,
                LearningSignalType.DISCOVERY_MADE
            ],
            'journey_tracking': [
                LearningSignalType.STATE_TRANSITIONED,
                LearningSignalType.OUTCOME_OBSERVED
            ],
            'verification': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.CALIBRATION_UPDATED
            ],
            'brand_intelligence': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.MECHANISM_ACTIVATED
            ],
            'ab_testing': [
                LearningSignalType.OUTCOME_OBSERVED
            ],
            'explanation_generator': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.MECHANISM_ACTIVATED
            ],
            'aot_reasoning': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.CALIBRATION_UPDATED,
                LearningSignalType.DISCOVERY_MADE
            ],
            'discovery_engine': [
                LearningSignalType.OUTCOME_OBSERVED,
                LearningSignalType.MECHANISM_ACTIVATED,
                LearningSignalType.TRAIT_INFERRED
            ]
        }
    
    async def start(self):
        """Initialize connections."""
        self.kafka_producer = aiokafka.AIOKafkaProducer(
            bootstrap_servers=self.kafka_bootstrap
        )
        await self.kafka_producer.start()
        
        self.redis = await aioredis.from_url(self.redis_url)
        
        self.neo4j = AsyncGraphDatabase.driver(
            self.neo4j_uri,
            auth=self.neo4j_auth
        )
    
    async def propagate(self, signal: LearningSignal):
        """
        Propagate a learning signal to all relevant components.
        
        Signals are routed based on:
        1. Component subscriptions (which types each component wants)
        2. Priority (immediate vs async vs batch)
        3. Target components (if specified, only those; else broadcast)
        """
        
        # Determine target components
        if signal.target_components:
            targets = signal.target_components
        else:
            # Broadcast to all subscribed components
            targets = [
                component 
                for component, subscriptions in self.component_subscriptions.items()
                if signal.signal_type in subscriptions
            ]
        
        signal.propagation_status = {t: 'pending' for t in targets}
        
        # Route based on priority
        if signal.priority == 'immediate':
            await self._propagate_immediate(signal, targets)
        elif signal.priority == 'fast':
            await self._propagate_fast(signal, targets)
        elif signal.priority == 'async':
            await self._propagate_async(signal, targets)
        else:  # batch
            await self._queue_for_batch(signal, targets)
    
    async def _propagate_immediate(
        self, 
        signal: LearningSignal, 
        targets: List[str]
    ):
        """
        Immediate propagation (<10ms).
        Used for: cache invalidation, hot prior updates.
        """
        
        tasks = []
        
        for target in targets:
            # Update Redis hot state
            tasks.append(
                self._update_redis_hot_state(signal, target)
            )
            
            # Invalidate affected caches
            tasks.append(
                self._invalidate_target_cache(signal, target)
            )
        
        await asyncio.gather(*tasks)
        
        for target in targets:
            signal.propagation_status[target] = 'immediate_complete'
    
    async def _propagate_fast(
        self, 
        signal: LearningSignal, 
        targets: List[str]
    ):
        """
        Fast propagation (<100ms).
        Used for: Kafka events, Redis updates, segment aggregates.
        """
        
        # Emit to Kafka topic
        kafka_message = {
            'signal_id': signal.signal_id,
            'signal_type': signal.signal_type.value,
            'source': signal.source_component,
            'payload': signal.payload,
            'timestamp': signal.timestamp.isoformat(),
            'targets': targets
        }
        
        await self.kafka_producer.send_and_wait(
            'adam.learning_signals',
            value=str(kafka_message).encode()
        )
        
        # Update segment-level aggregates
        if 'user_id' in signal.payload:
            await self._update_segment_aggregates(signal)
        
        for target in targets:
            signal.propagation_status[target] = 'fast_complete'
    
    async def _propagate_async(
        self, 
        signal: LearningSignal, 
        targets: List[str]
    ):
        """
        Async propagation (<1s).
        Used for: Neo4j writes, credit attribution, calibration updates.
        """
        
        # Write to Neo4j
        await self._write_signal_to_neo4j(signal)
        
        # Run credit attribution
        if signal.signal_type == LearningSignalType.OUTCOME_OBSERVED:
            await self._run_credit_attribution(signal)
        
        # Update calibration if needed
        if signal.signal_type in [
            LearningSignalType.OUTCOME_OBSERVED,
            LearningSignalType.CALIBRATION_UPDATED
        ]:
            await self._update_calibration(signal)
        
        for target in targets:
            signal.propagation_status[target] = 'async_complete'
    
    async def _queue_for_batch(
        self, 
        signal: LearningSignal, 
        targets: List[str]
    ):
        """
        Queue for batch processing (hourly/daily).
        Used for: Causal inference, cluster updates, embedding retraining.
        """
        
        # Add to batch queue in Redis
        await self.redis.rpush(
            'adam:batch_signals',
            str({
                'signal': signal.__dict__,
                'targets': targets,
                'queued_at': datetime.utcnow().isoformat()
            })
        )
        
        for target in targets:
            signal.propagation_status[target] = 'batch_queued'
    
    # -------------------------------------------------------------------------
    # Specific Propagation Handlers
    # -------------------------------------------------------------------------
    
    async def _update_redis_hot_state(
        self, 
        signal: LearningSignal, 
        target: str
    ):
        """Update hot state in Redis based on signal type."""
        
        if signal.signal_type == LearningSignalType.OUTCOME_OBSERVED:
            user_id = signal.payload.get('user_id')
            mechanism_id = signal.payload.get('mechanism_id')
            outcome = signal.payload.get('outcome')
            
            if user_id and mechanism_id:
                # Update user × mechanism success rate
                key = f"adam:hot:{user_id}:{mechanism_id}"
                
                current = await self.redis.hgetall(key)
                successes = int(current.get(b'successes', 0))
                failures = int(current.get(b'failures', 0))
                
                if outcome:
                    successes += 1
                else:
                    failures += 1
                
                await self.redis.hset(key, mapping={
                    'successes': successes,
                    'failures': failures,
                    'rate': successes / (successes + failures),
                    'updated': datetime.utcnow().isoformat()
                })
                await self.redis.expire(key, 300)  # 5 min TTL
    
    async def _invalidate_target_cache(
        self, 
        signal: LearningSignal, 
        target: str
    ):
        """Invalidate caches affected by this signal."""
        
        user_id = signal.payload.get('user_id')
        
        if user_id:
            # Invalidate user-specific caches
            patterns = [
                f"adam:cache:{target}:{user_id}:*",
                f"adam:inference:{user_id}:*"
            ]
            
            for pattern in patterns:
                keys = await self.redis.keys(pattern)
                if keys:
                    await self.redis.delete(*keys)
    
    async def _write_signal_to_neo4j(self, signal: LearningSignal):
        """Persist signal to Neo4j for permanent record."""
        
        query = """
        MERGE (ls:LearningSignal {signal_id: $signal_id})
        SET ls.signal_type = $signal_type,
            ls.source = $source,
            ls.timestamp = $timestamp,
            ls.payload = $payload
        
        WITH ls
        
        // Connect to user if applicable
        OPTIONAL MATCH (u:User {user_id: $user_id})
        FOREACH (_ IN CASE WHEN u IS NOT NULL THEN [1] ELSE [] END |
            MERGE (u)-[:GENERATED]->(ls)
        )
        
        // Connect to mechanism if applicable
        OPTIONAL MATCH (m:CognitiveMechanism {mechanism_id: $mechanism_id})
        FOREACH (_ IN CASE WHEN m IS NOT NULL THEN [1] ELSE [] END |
            MERGE (ls)-[:ABOUT]->(m)
        )
        
        RETURN ls
        """
        
        async with self.neo4j.session() as session:
            await session.run(
                query,
                signal_id=signal.signal_id,
                signal_type=signal.signal_type.value,
                source=signal.source_component,
                timestamp=signal.timestamp.isoformat(),
                payload=str(signal.payload),
                user_id=signal.payload.get('user_id'),
                mechanism_id=signal.payload.get('mechanism_id')
            )
    
    async def _run_credit_attribution(self, signal: LearningSignal):
        """
        Attribute the outcome to the decisions that led to it.
        
        This enables the system to learn WHICH DECISIONS were good,
        not just whether the overall interaction was successful.
        """
        
        user_id = signal.payload.get('user_id')
        session_id = signal.payload.get('session_id')
        outcome = signal.payload.get('outcome')
        
        if not all([user_id, session_id, outcome is not None]):
            return
        
        # Find all decisions in this session
        query = """
        MATCH (s:Session {session_id: $session_id})-[:CONTAINS]->(d:Decision)
        MATCH (d)-[:USED_MECHANISM]->(m:CognitiveMechanism)
        MATCH (d)-[:FOR_USER]->(u:User {user_id: $user_id})
        
        RETURN d.decision_id, d.confidence, d.mechanism_weights,
               collect(m.mechanism_id) as mechanisms
        ORDER BY d.timestamp
        """
        
        async with self.neo4j.session() as session:
            result = await session.run(
                query,
                session_id=session_id,
                user_id=user_id
            )
            decisions = await result.data()
        
        # Attribute credit using temporal difference learning
        # More recent decisions get more credit (recency weighting)
        # Higher confidence decisions get more credit (confidence weighting)
        
        total_decisions = len(decisions)
        
        for i, decision in enumerate(decisions):
            # Recency weight: later decisions matter more
            recency_weight = (i + 1) / total_decisions
            
            # Confidence weight: higher confidence = more responsibility
            confidence_weight = decision['confidence']
            
            # Combined credit
            credit = recency_weight * confidence_weight
            
            # Update each mechanism involved
            for mechanism_id in decision['mechanisms']:
                update_query = """
                MATCH (u:User {user_id: $user_id})-[r:RESPONDS_TO]->(m:CognitiveMechanism {mechanism_id: $mechanism_id})
                SET r.alpha = CASE WHEN $outcome THEN r.alpha + $credit ELSE r.alpha END,
                    r.beta = CASE WHEN NOT $outcome THEN r.beta + $credit ELSE r.beta END,
                    r.total_observations = r.total_observations + 1,
                    r.last_observation = datetime()
                """
                
                async with self.neo4j.session() as session:
                    await session.run(
                        update_query,
                        user_id=user_id,
                        mechanism_id=mechanism_id,
                        outcome=outcome,
                        credit=credit
                    )
    
    async def _update_calibration(self, signal: LearningSignal):
        """Update confidence calibration based on outcome."""
        
        predicted_confidence = signal.payload.get('predicted_confidence')
        actual_outcome = signal.payload.get('outcome')
        
        if predicted_confidence is None or actual_outcome is None:
            return
        
        # Add to calibration data
        bucket = int(predicted_confidence * 10)  # 0-10 buckets
        
        key = f"adam:calibration:bucket:{bucket}"
        
        await self.redis.hincrby(key, 'predictions', 1)
        if actual_outcome:
            await self.redis.hincrby(key, 'positive_outcomes', 1)
        
        # Periodically recalculate calibration curve
        predictions = int(await self.redis.hget(key, 'predictions') or 0)
        if predictions % 100 == 0:  # Every 100 predictions
            await self._recalculate_calibration()
    
    async def _update_segment_aggregates(self, signal: LearningSignal):
        """Update segment-level statistics."""
        
        user_id = signal.payload.get('user_id')
        
        # Get user's cluster
        query = """
        MATCH (u:User {user_id: $user_id})
        RETURN u.psychological_cluster_id as cluster
        """
        
        async with self.neo4j.session() as session:
            result = await session.run(query, user_id=user_id)
            record = await result.single()
        
        if record:
            cluster_id = record['cluster']
            mechanism_id = signal.payload.get('mechanism_id')
            outcome = signal.payload.get('outcome')
            
            if mechanism_id and outcome is not None:
                # Update cluster × mechanism effectiveness
                key = f"adam:segment:{cluster_id}:{mechanism_id}"
                
                await self.redis.hincrby(key, 'total', 1)
                if outcome:
                    await self.redis.hincrby(key, 'successes', 1)
```

# Part 3: Structural Moats Through Architecture

## 3.1 The Moat Philosophy: Why Architecture Beats Algorithms

Competitors can copy algorithms. They can hire engineers. They can license data.

What they **cannot** replicate is **structural advantage that compounds over time**.

ADAM's architecture creates three types of moats:

### Moat Type 1: The Graph Flywheel

```
More users → More interactions → Richer graph relationships →
Better predictions → Higher conversion → More advertisers →
More budget → More users → ...
```

The graph doesn't just get *bigger*—it gets *smarter*. Each relationship encodes learned intelligence. The 10 millionth user benefits from everything learned from the first 9,999,999.

### Moat Type 2: Emergent Psychological Knowledge

The system discovers psychological patterns that don't exist in any textbook. These emergent constructs are proprietary knowledge—they exist only in ADAM's graph.

A competitor starting fresh would need to:
1. Collect the same volume of interaction data
2. Build the same discovery infrastructure
3. Wait for the same patterns to emerge

This takes **years**, not months.

### Moat Type 3: Cross-Component Learning Density

Every component learns from every other component. This creates combinatorial advantage—the value isn't in any single component but in their **interconnections**.

```
N components → N² potential learning pathways → 
Exponential advantage over isolated systems
```

```python
        'moats_that_create_proprietary_knowledge': sum(1 for m in moats if m.creates_proprietary_knowledge),
        'moats_uncopyable_with_money': sum(1 for m in moats if not m.can_be_copied_with_money)
    }
```

# Appendix B: Key Neo4j Indexes for Discovery

```cypher
// =============================================================================
// INDEXES FOR DISCOVERY QUERIES
// These enable serendipitous pattern finding, not just efficient retrieval
// =============================================================================

// ----- Entity indexes -----
CREATE INDEX user_cluster IF NOT EXISTS
FOR (u:User) ON (u.psychological_cluster_id);

CREATE INDEX construct_framework IF NOT EXISTS
FOR (c:PsychologicalConstruct) ON (c.theoretical_framework);

CREATE INDEX mechanism_theoretical IF NOT EXISTS
FOR (m:CognitiveMechanism) ON (m.is_theoretically_derived);

CREATE INDEX signature_mapped IF NOT EXISTS
FOR (bs:BehavioralSignature) ON (bs.is_mapped_to_construct);

CREATE INDEX signature_correlation IF NOT EXISTS
FOR (bs:BehavioralSignature) ON (bs.outcome_correlation);

CREATE INDEX emergent_review IF NOT EXISTS
FOR (ec:EmergentConstruct) ON (ec.requires_theoretical_review);

CREATE INDEX anomaly_status IF NOT EXISTS
FOR (a:Anomaly) ON (a.investigation_status);

CREATE INDEX hypothesis_status IF NOT EXISTS
FOR (h:Hypothesis) ON (h.status);

// ----- Relationship indexes for learning queries -----
CREATE INDEX responds_to_rate IF NOT EXISTS
FOR ()-[r:RESPONDS_TO]-() ON (r.success_rate);

CREATE INDEX responds_to_observations IF NOT EXISTS
FOR ()-[r:RESPONDS_TO]-() ON (r.total_observations);

CREATE INDEX interacts_with_effect IF NOT EXISTS
FOR ()-[i:INTERACTS_WITH]-() ON (i.interaction_effect);

CREATE INDEX predicts_correlation IF NOT EXISTS
FOR ()-[p:PREDICTS_RESPONSE]-() ON (p.correlation);

// ----- Full-text indexes for hypothesis search -----
CREATE FULLTEXT INDEX hypothesis_search IF NOT EXISTS
FOR (h:Hypothesis) ON EACH [h.statement, h.theoretical_basis];

CREATE FULLTEXT INDEX construct_search IF NOT EXISTS
FOR (c:PsychologicalConstruct) ON EACH [c.name, c.theoretical_definition];

// ----- Composite indexes for discovery queries -----
CREATE INDEX signature_discovery IF NOT EXISTS
FOR (bs:BehavioralSignature) 
ON (bs.outcome_correlation, bs.is_mapped_to_construct, bs.population_prevalence);
```
